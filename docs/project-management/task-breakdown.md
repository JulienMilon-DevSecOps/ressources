# 🧱 Task Breakdown Guide

## 🎯 Objectif
Découper les fonctionnalités en tâches simples et réalisables.

---

## 📌 Pourquoi découper ?

- Facilite le développement
- Améliore la lisibilité
- Permet un meilleur suivi

---

## 🧩 Exemple

Feature : Authentification

Découpage :
- [ ] Créer endpoint login
- [ ] Implémenter JWT
- [ ] Ajouter middleware auth
- [ ] Tests

---

## ⚠️ Bonnes pratiques

- On peut créer une issue pour la tache globale, cette tache sera notée en Epic ou User Story suivant la formulation(groupement de tache) et les sous-tâches seront des features principalement
- Tâches < 1 jour
- Indépendantes
- Testables

---

## ❌ À éviter

- Tâches trop larges -> Erreurs ou uncompréhension car sujet à interprétation car pas assez précis
- Dépendances complexes
